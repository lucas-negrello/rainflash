## Desenvolvimento das Features de Users, Companies e Roles

### Users

- [ ] Criar o resource de `Users`.
- [ ] Implementar o resource de `Users` com os padrões da aplicação,
    utilizando a pasta `Shared` para itens em comum.
- [ ] Implementar a funcionalidade de atribuição de `Roles` e `Permissions` aos `Users`.

### Companies

- [ ] Criar o resource de `Companies`.
- [ ] Implementar o resource de `Companies` com os padrões da aplicação,
    utilizando a pasta `Shared` para itens em comum.
- [ ] Implementar o `RelationManager` de `Users` dentro do resource de `Companies`.
- [ ] Permitir que a `Company` veja apenas os `Users` associados a ela e só possa associar `Users` criados por ela.

### CompanyUsers e CompanyUserRoles

- [ ] Criar o `RelationManager` de `CompanyUsers` dentro do resource de `Companies`.
- [ ] Criar o `RelationManager` de `CompanyUsers` dentro do resource de `Users`.
- [ ] Criar o `RelationManager` de `CompanyUserRoles` dentro dos resources de `Users` e `Companies`.
